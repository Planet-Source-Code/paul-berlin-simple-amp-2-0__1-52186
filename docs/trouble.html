Troubleshooting


Here are solutions to most problems.

Problem: Stuttering/Skipping sound.

  • Update your soundcard drivers to the newest available.
  • Update your video card drivers.
  • Increase sound buffer.
  • Disable DSP.
  • Disable DirectX Effects.
  • Select the most appropriate Mixer for your system (see info in settings window).
  • Reduce Sampling Frequency.
  • If playing modules, try reducing Max Channels.
  • Try another Output Engine.

    • Problem: I am told that I am missing files when I start Simple Amp.

  • Try reinstalling simple amp.
  • Get the missing file from the internet.

    • Problem: The File Browser is so damn slow...

  • The first time you browse an file, it will take an while, but later it should go alot faster.
  • Make sure Media Library is enabled.
  • Avoid browsing files over slow network connections.
  • VBR (Variable Bitrate) MP3s are slower than ordinary mp3s when browsing them for the first time.

    • Problem: Slow Visualization.

  • Decrease/Increase Update speed.
  • Turn off Fade or select an faster fade speed.
  • Use an skin with smaller visualization area.
  • Choose an less CPU intensive visualization (like volume meters).
  • Turn off the visualization.

    • Problem: Overall slowness.

  • Update Audio/Video drivers.
  • Update your computer.

    • Problem: Opening/browsing some modules chrashes the program.

  • Do not open or browse them. FMOD cannot open some modules, and for some reason crashes. This is out of my control.

    • Problem: FMOD Init Error when starting.

  • Try restarting before you do anything.
  • Select an other mixer/output engine/soundcard and restart.
  • If DirectX effects is on, increase buffer to 100ms or over.
  • Delete Settings.ini and try again, with autodetect & default settings.

    • Problem: Surround sound has no hearable effect.

  • Surround sound does not make as big an effect on all songs, try an other one.
  • Surround sound only (for now) works on CPU/FPU mixer.
  • Surround sound does not work on some music types.
  • Surround sounds best on headphones or stereo speakers.

    • Problem: Media Library is slow.

  • Perform Maintenance. In Settings/Media Library tab/Library Maintenance.

    • Problem: The next song in the playlist doesn't play.

  • Turn on auto advance. Press A when focus is on the main player window.
  • You have reached the end of your playlist. Try turning on repeat and press next.

    • Problem: The wrong covers are shown in the File Info window.

  • Name the cover images to '[Artist] - [Album] [Front/Back]' for best performance. Also note that only .BMP, .GIF & .JPG images can be used.
  • This is just an guess by the program, do not keep images other than covers in the same folder as your music for best performance.

    • Problem: Playing CD Audio works bad or songs are skipped.

  • Well, you're right.

    • Problem: I want to play an Audio CD from an other drive than the default drive.

  • Too bad, you can't.
  • Is it so hard to just insert the disc in your other drive?

    • Problem: There are several small gnomes jumping around on screen whenever I have Simple Amp in focus.

  • You can press CTRL+L to toggle them on/off.
  • Press CTRL+I to make them scream insults.

    Known Bugs/Problems


    Here are some known bugs. If you find any other bug and you can reproduce it, please mail me about it.

  • Sometimes Media Library seems to load with strange or empty information. (May be fixed already)
  • In File Viewer, it says that the song has been played and skipped alot of times (like 16000, or -16000). (May be fixed already)
  • Media Library sometimes reports errors when quitting Simple Amp about not able to save media library. (This may be fixed already)
  • Using network paths like //localhost may not work correct with certain parts of the program (like wehn saved in .playlist files). Binding your network paths to a drive will solve this, though.
  • Some mp3s with ID3v2 tags do not read correctly with the ID3v23x.DLL. This seems to be a problem with the mp3 tags having a wrong length value for the comments string. But they work fine in WinAmp so I don't know...
  • The method of determing if mp3s are VBR is stupid (look in modMisc) and does not always work.

    • Look in Programming Notes for more bug related info.
