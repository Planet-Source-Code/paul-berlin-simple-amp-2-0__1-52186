Programming Notes


Because this program started in early 2002 (now it's september 2003) and it has not been fully rewritten, some parts of the code are older than other parts. Thus some parts are written better than other, some are more commented than other and some are actually just a mess. It may be hard to find your way through the 18000 or so lines of code =).

Feel free to change/add/improve this code, but if you do it would be nice if you send me the updated code =).

You are free to use this code in your own programs, as long as you give credit where credit is due.

Check out troubleshooting for some known bugs.

I will probably NOT make any more major versions, but here are some things I would add/fix:

What needs fixing/improving/adding and other ideas

  • A complete rewriting would be nice, to clean up, speed up, etc.
  • Frameskip for visualization. Currently, peaks dropspeed etc. is set in values per frame. Thus the actual speed of the peaks is different in lower visualization update speeds than in higher. So a drop speed of 10 values per frame with visualization update speed at 50 ms (new frame every 50 ms) that ammounts to 200 values per second. When the update speed is 25 ms it is 400 values per second.
  • Cleanup/rewrite the code for moving the player window with with playlist docked and screen snap on playlist when playlist is docked to player.
  • Perhaps adding music crossfading possibilities to clsFmod.
  • Perhaps some more effects in clsImg, for example blur.
  • Better systray class, for better control and WinXP 'bubble' tooltips.
  • Reading tags from WMA.
  • Lables that scroll their content if it does not fit.
  • Remake the entire playlist list control to be able to show folders of items. For example one folder for each artist in the list. You would of course be able to open/close the folders.
  • Better toolbar controls (in frmAdd) so I can use one control with all buttons, with text label to the right of the buttons without all the buttons being forced the same size.
  • Icons in the list in frmAdd (associated icons).
  • New and better skin format. Complete control over everything, scripting, PNG-images, resizable playlist (to any size), etc.
  • Remove current visualization system (or perhaps simplify it) and add one more similar to Winamps AVS, using perhaps DirectX.
  • Perhaps render everyting in DirectX for superfast alpha effects & fades
  • Ability for user to enter other information for each file in Media Library, such as Mood, Rating etc. Or perhaps just add it to the ID3v2 tag.
  • Reduce the large memory usage.
  • Rewrite in Visual C++ instead!
  • Come up with a better name! Complex Amp =)
  • Only your imagination is the limit! (well sort of)
